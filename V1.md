# V1.md — Hawksnest road to a 1.0 release

Status doc for getting Hawksnest from "Phase 4 complete, feels good in daily use" to a
defensible **v1.0**. Companion to [ROADMAP.md](ROADMAP.md) (the departing-engineer backlog) —
this file is the sequenced plan with live status. Convention borrowed from `Magpie/V1.md`.

Last updated: 2026-07-11.

## Where things stand

The app is in good shape: zero TODO/FIXME debt, invariants documented, CI green on main. Baseline
health as of this writing — **228→230 vitest, 27/27 Playwright e2e, Android unit tests + Sift
gate green** (the "Sift design audit" CI line is red but **non-gating** and mis-firing — see
Follow-ups). What's shipped toward v1 already:

- **[merged? PR #50] Android control feel + reliability** — `ControlGate` (crash-safe control
  calls + honest pending), `SlideToAct` locks, optimistic light/switch/fan toggles, per-segment
  alarm spinners, semantic haptics. Fixed the uncaught-`callService` crash (any control tap while
  disconnected used to kill the app).
- **[open PR #51] Web control parity** — alarm arm/disarm made non-optimistic with pending +
  error (dashboard `SecurityStatusBar` + `AlarmCard`, shared `useAlarmControl`); `LockCard`
  no-echo timeout.
- **[open PR #52] mock-ha port parameterized** (`MOCK_HA_PORT`) so the e2e suite runs on the
  Dragonfly host, where 8765 collides with the kidbot container.

## The gates (do in order; each is a shippable slice)

### Gate 1 — correctness  ✅ substantially done

1. ✅ Android: crash-safe control calls + pending/disabled state for lock & alarm (PR #50).
2. ✅ Web: AlarmCard/SecurityStatusBar pending + error; LockCard no-echo timeout (PR #51).
3. ✅ Parameterize the mock-ha port (PR #52).

### Gate 2 — security posture (the app unlocks doors; this outranks features)

4. **TLS on the proxy → kill cleartext.** Front the k3s nginx / Tailscale path with TLS
   (Tailscale Serve gives free HTTPS at a MagicDNS name), standardize on the hostname, then flip
   Android `network_security_config.xml` `cleartextTrafficPermitted="false"`. Closes the known,
   accepted Phase-0 hole. (ROADMAP #1.)
5. **Decide the token-in-cloud-backup question.** The Android LLAT (a full house credential,
   incl. locks) is included in Google cloud backup + device-transfer by
   `backup_rules.xml`/`data_extraction_rules.xml`, and `CredentialStore` is unencrypted DataStore.
   Pick one: (a) exclude from backup (re-login on a new phone), or (b) Keystore-wrap the token.
   **Owner decision needed** — see Open decisions.
6. **OAuth to HA** — replace the long-lived token with HA's OAuth + refresh, web + Android
   together so the token story stays one story. (ROADMAP #2.) The largest single item here; see
   Open decisions on whether it blocks v1.

### Gate 3 — the missing capability

7. **Push notifications** (Android Phase 4). A doorbell app that only alerts while open isn't a
   doorbell app. The in-app banner already keys off the `_ding` sensor; push needs a server-side
   hook (HA automation → relay) + transport. **Suite-level decision** (ntfy vs FCM) — build once
   for all suite apps, per the suite ROADMAP Tier-3 push item. v1 payload: doorbell ding + alarm
   state changes.

### Gate 4 — polish + release

8. **PWA update toast** — a "new version, reload" prompt when the service worker swaps the shell
   (silent updates strand a stale tab on the wall-tablet use case). (ROADMAP #6.)
9. **Light theme** — the tokens layer was built for it; port the contrast-safe light variants
   from the Pulse reference palette. (ROADMAP #5.)
10. **Camera live-path smoke checklist** — the WebRTC (web) / LL-HLS (Android) live path has no
    automated coverage (mock serves no `web_rtc` camera). A written per-release checklist ("live
    paints < 3 s, event scrub works, doorbell banner fires") formalizes what's tribal. (ROADMAP #4.)
11. **Cut 1.0** — close stale PR #17, merge Dependabot, bump Android `VERSION_NAME` base
    `0.1`→`1.0` (build.gradle.kts) and web `package.json` `0.0.0`→`1.0.0`. Pushing to main cuts
    `android-v1.0.x`.

## Open decisions (need the owner)

- **Does OAuth (item 6) block v1, or land in v1.1?** Recommendation: with TLS done (item 4) and
  the LLAT being revocable, ship **v1.0 after Gate 2 items 4–5** and make OAuth the headline of
  **v1.1**, rather than blocking the release on the biggest chunk.
- **Push transport (item 7): self-hosted ntfy vs FCM relay?** ntfy fits the suite's local-first
  bias and also covers Dragonfly's "update available" nudges — recommend ntfy, built once at the
  suite level.
- **Token-at-rest (item 5): exclude-from-backup vs Keystore-wrap?**

## Follow-ups (tracked, not blocking)

- **Sift audit is mis-firing.** The advisory `android-ci.yml` Sift job (`continue-on-error: true`,
  non-gating) fails at Gradle *configuration*, not on findings: the sibling **Sift repo is still on
  AGP 8.5.0** while Hawksnest moved to **9.1.1** (Tier-2 toolchain bump), and a composite build
  can't mix AGP majors. So the design audit hasn't actually run since the bump. Fix is in the
  `CDRaab01/Sift` repo (bump its AGP/Gradle to match), not here.
- **Stale PR #17** ("cut a signed release APK") — superseded by `android-release.yml`; close it.
